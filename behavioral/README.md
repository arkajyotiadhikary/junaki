# Behavioral Interview Track

A bank of real-work stories for behavioral rounds. Amazon weights its **16 Leadership Principles** so heavily that a bar raiser can veto an offer on behavioral signal alone; Meta runs a dedicated behavioral round on its core values. The same well-structured story, told from your actual work, answers both.

## The STAR(L) format

Every story is structured so you can tell it in ~2 minutes and survive follow-up probes:

- **S — Situation**: the context. Where, when, your role, the stakes. Brief.
- **T — Task**: *your* specific responsibility or goal. Use "I", not "we". What made it hard.
- **A — Action**: the heart of it — the concrete steps **you** took and the trade-offs you weighed. This is where the Leadership Principles show up. Spend the most time here.
- **R — Result**: the outcome, quantified (numbers, %, time, incident resolved). Tie it back to the Task.
- **L — Learning**: what you took away and how it changed how you work. The letter most candidates skip — it's what signals growth and self-awareness.

Drill a story out loud, then log it through the existing spaced-repetition loop:

```
npm run practice -- behavioral/stories/<slug>.md --rating=<fail|hard|good|easy>
```

The rating drives the same interval ladder as coding problems, so stories resurface in `npm run today`. (The `strength: 1-5` field in each story's frontmatter is a *separate* hand-edited self-assessment — not the drill rating.)

Scaffold a new story from the template:

```
npm run story -- <slug> "<Story Title>"
```

Regenerate the coverage matrix below from every story's frontmatter:

```
npm run story:matrix
```

## Suggested 12-story plan

A starter set of archetypes chosen so that, once written, every Leadership Principle and theme is covered by at least 2 stories. Swap in your own real specifics — scaffold each with `npm run story -- <slug> "<Title>"`.

| # | Slug | Story (your real version of…) | Principles | Themes |
|---|------|-------------------------------|------------|--------|
| 1 | `outage-ownership` | Owned a production outage end to end | OW, DD, BA, ET, DR | failure, deadline, technical |
| 2 | `disagreed-mgr` | Disagreed with your manager, then committed | BB, ET, RA | conflict, influence |
| 3 | `simplified-pipeline` | Simplified an over-engineered system | IS, HS, FR | technical, ambiguity |
| 4 | `missed-deadline` | Missed a deadline and recovered | OW, DR, ET | failure, deadline |
| 5 | `cross-team-migration` | Drove a cross-team change with no authority | TB, ET, DR | influence, ambiguity |
| 6 | `hard-feedback` | Received hard feedback and changed | LC, ET, EB | failure, conflict |
| 7 | `incomplete-data` | Made a call with incomplete data | BA, RA, OW | ambiguity, deadline |
| 8 | `mentored-teammate` | Grew a teammate / raised the team | HD, EB, HS | influence |
| 9 | `customer-bug-deep-dive` | Dug deep into a customer-impacting bug | CO, DD, OW, LC | technical, failure |
| 10 | `raised-quality-bar` | Held the quality bar under ship pressure | HS, BB, ET, HD | conflict, deadline |
| 11 | `ambitious-bet` | Proposed an ambitious long-term project | TB, IS, BR | ambiguity, influence |
| 12 | `infra-cost-scale` | Cut cost / scaled responsibly | FR, BR, OW, CO | technical, deadline |

See `questions.md` for 30 practice prompts mapped to these principles.
